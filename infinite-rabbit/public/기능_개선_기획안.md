## 1. 개요

### 1-1. 목적

이 문서의 목적은 로컬 스크래치 폴더에 구현된 웹 미니게임 「아기 토끼 무한의 계단」의 개선 과제 범위, 정책, 화면·플로우를 한곳에 정리하여 개발·검증·배포 시 기준으로 삼는 것이다.

### 1-2. 배경·문제

현재 실행은 주로 브라우저에서 로컬 파일 URL로 연다.  
**`file:///C:/Users/HANA/.gemini/antigravity/scratch/infinite-rabbit/index.html`**  
이 방식은 별도 서버 없이 바로 플레이할 수 있으나, `game.js`의 스프라이트 배경 투명 처리(`removeWhiteBg`, Canvas `getImageData`)가 브라우저 보안 정책(CORS)에 의해 동작하지 않을 수 있어, 콘솔 경고 또는 `file://` 전용 안내 `alert`가 발생할 수 있다.

PC는 방향키·A/D, 모바일은 하단 좌·우 버튼으로 조작 가능하나, 일시정지·설정·사운드·랭킹 등은 미구현이며 난이도는 점수·타이머 소모율에 연동되어 있어 플레이 테스트를 통한 구간별 밸런스 조정이 필요하다.

## 3. 주요 내용 · 개선 사항

### 3-1. 로컬 실행 환경·안내 문구 정비

2026년 4월 기준 사용자는 **`file:///C:/Users/HANA/.gemini/antigravity/scratch/infinite-rabbit/index.html`** 로 즉시 실행할 수 있으나, 배경 투명 처리 실패 시 게임 내 `alert`로 HTTP 플레이를 유도하고 있다. 시작 화면·README 수준에서 두 방식의 차이를 짧게 안내하고, 가능하면 로컬 서버 한 줄 실행 예시를 고정 문구로 둔다.

**상세 내용**

- 에셋 로드 완료 후에만 시작 버튼을 `게임 시작`으로 전환하는 현행 동작 유지 (`processAssets`)
- `file://`에서 CORS로 투명 처리가 막힐 때의 메시지를 사용자 행동(탭 주소 변경 또는 서버 실행) 중심으로 다듬기
- (선택) 사내·공개 정적 호스팅 URL을 「권장 주소」로 명시해 `file://` 의존도 완화

### 3-2. 난이도·보상(타이머·계단·당근) 밸런스

점수가 오를수록 시간 소모가 빨라지고(`drainRate`), 성공 시 회복량은 점수에 따라 줄어드는 구조이므로, 초반 학습 구간과 고득점 구간의 체감 난이도를 목표 플레이 시간에 맞게 조정한다.

**상세 내용**

- `maxTime`, `drainRate`, 회복 계수(`recovery`), 계단 수평·수직 간격, 당근 스폰 확률을 코드 상수 또는 설정 객체로 모아 튜닝 비용 절감
- 당근 획득 시 UI(당근 수) 반영 및 이모지 표시 정책 유지 여부 검토

### 3-3. 입력·UI·접근성

웹·모바일 브라우저 공통으로 하단 좌·우 버튼(`touchstart`)과 키보드 입력을 유지하고, 오터치·가림을 줄이기 위한 버튼 크기·여백과 `#controls` 노출 타이밍(플레이 중만)을 점검한다.

**상세 내용**

- 상단 점수·타이머 바 가독성(폰트·그림자·잔량 30% 미만 시 색상 변화) 유지
- (선택) `prefers-reduced-motion` 등 단순 연출 축소 옵션 검토

### 4-2. 흐름도 / 시퀀스

고객이 **`file:///C:/Users/HANA/.gemini/antigravity/scratch/infinite-rabbit/index.html`** (또는 로컬 HTTP 동일 페이지)로 진입한다.

페이지가 로드되면 `loadAssets()`가 배경·토끼·계단 이미지를 읽고, 모두 완료되면 `processAssets()`에서 시작 버튼에 「게임 시작」을 표시한다.

고객이 「게임 시작」을 누르면 `initGame()`으로 계단·플레이어·카메라가 초기화되고 플레이 상태가 된다.

플레이 중 좌·우 입력 시 `jump(dir)`가 호출되어 다음 계단 방향과 일치하면 점수·타이머 회복·필요 시 계단 추가가 수행되고, 불일치하면 게임 오버 상태로 전환된다.

타이머가 0이 되어도 게임 오버가 되며, 낙하 연출 후 화면에 최종 점수·당근이 표시되고 「다시 하기」로 재시작할 수 있다.
